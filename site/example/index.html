    Example Walkthrough

    This section demonstrates how Manthan produces deterministic decisions using real scenarios.

    Example 1 — High Transaction Risk

    Input

    {
  "amount": 15000,
  "country": "high_risk",
  "user_verified": false
}

    Rules

    IF amount > 10000 → reject
IF country == high_risk → reject

    Decision Flow

    • amount > 10000 → TRUE
    • First match → reject

    Output

    {
  "decision": "reject",
  "reason": "amount threshold exceeded"
}

    Example 2 — Safe Transaction

    Input

    {
  "amount": 2000,
  "country": "low_risk",
  "user_verified": true
}

    Rules

    IF amount > 10000 → reject
ELSE → approve

    Decision Flow

    • amount > 10000 → FALSE
    • No reject rule matched → approve

    Output

    {
  "decision": "approve",
  "reason": "within safe limits"
}

    Example 3 — Decision Graph Execution

    Input

    {
  "user_age": 17,
  "document_verified": false

    Graph

    • user_age < 18 → not eligible

    {
  "decision": "reject",
  "reason": "age and verification failure"
}

    Example 4 — Contract-Based Decision

    Contract (v1.0)

    rules:
  - condition: "score < 50"
    action: "reject"
  - condition: "score >= 50"
    action: "approve"

    Input

    {
  "score": 72
}

    Evaluation

    • score < 50 → FALSE
    • score >= 50 → TRUE

    Output

    {
  "decision": "approve",
  "contract_version": "v1.0"
}

    Determinism Proof

    For ALL examples:

    Same Input → Same Rules → Same Output

    No randomness
    No ambiguity
    No hidden state

    Key Takeaway

    Manthan converts:

    Inputs → Rules → Decisions

    Into a fully deterministic system.
